Fleet Backup

A backup server stores the archives, a repository is a named place inside it, and a job writes one machine's paths into a repository every night. Build them in that order. Vaier emails admins when a run fails, and when the backup server goes quiet.

  1. 1

    Stand up a backup server

    The machine that stores the fleet's archives — often a NAS. Vaier reaches it over SSH to install borg and to trust the machines that back up to it. You designate the backup server in the Explorer; provision it and use it from here.

  2. 2

    Name a repository on it

    A repository is where one machine's archives land. Name it and Vaier derives its path on the server, generates a passphrase, and creates it on the first run.

  3. 3

    Give a machine a job

    Which paths to back up, into which repository, and how long to keep them. Check readiness to install borg on the machine and trust its key, then run it on demand or leave it to the nightly schedule.

  4. Every night, on its own

    Enabled jobs run once a day at this hour, in the server's zone. A failed run emails all admins.
